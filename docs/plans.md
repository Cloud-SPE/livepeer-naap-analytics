# Plans

Execution plans are first-class artifacts in this repository.

## Canonical Open Work

Open work is tracked through the pair below:

- [`backlog.md`](backlog.md) for current engineering and product-facing work
- [`exec-plans/tech-debt-tracker.md`](exec-plans/tech-debt-tracker.md) for residual implementation debt and cleanup follow-up

Use this page to navigate execution plans around that open-work pair.

## Active Plans

See [`exec-plans/active/`](exec-plans/active/).

## Completed Plans

See [`exec-plans/completed/`](exec-plans/completed/).

## Notes

- Create an exec plan before substantial multi-step work.
- Keep active plans decision-complete and outcome-oriented.
- Do not use active plans as a second backlog. When work is closed, move the plan to `completed/`.
- Historical sequencing notes that are no longer part of the active workflow are preserved outside the active docs surface.
